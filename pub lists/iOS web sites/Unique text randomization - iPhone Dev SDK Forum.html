Unique text randomization
I'm just beginning iPhone development and I'm working on an app that will display text randomly without any repeats. So far I have the random part down but I can't seem to figure out a way to not have any repeats. Any help would be appreciated.

You could put all of your strings in an NSMutableArray, and then remove them as you pick them at random.

Code:
NSMutableArray *myLines;

//load the strings from a plist file into a mutable array.
//only do this once at the start, and again when the array is empty
myLines = [[NSMutableArray alloc] initWithContentsOfFile:path];

//choose a number, get the line, remove the line
int chosenNumber = arc4random() % [myLines count];
NSString *chosenLine = [myLines objectAtIndex:chosenNumber];
[myLines removeObjectAtIndex:chosenNumber];
I switched to arc4random so you'll get a different order every time; it seeds itself.

That looks good - except that you're loading a new full array every time the method is called, so you're not getting the no-repeat effect you wanted. You need to declare line10Array outside of this method, and load the array only when the old array is empty.

Code:
//declare in the .h file
NSMutableArray *line10Array;

- (IBAction)goto10Again {

	if ([line10Array count]==0){
		
		[line10Array release]; // release old array; it's empty		

		// Path to the plist (in the application bundle)
		NSString *path = [[NSBundle mainBundle] pathForResource:@"line10" ofType:@"plist"];
		// Build the array from the plist  
		line10Array = [[NSMutableArray alloc] initWithContentsOfFile:path];
	}

	int chosenNumber = arc4random() % [line10Array count];
	NSString *chosenLine = [line10Array objectAtIndex:chosenNumber];
	[line10Array removeObjectAtIndex:chosenNumber];
        [textview10 setText:chosenLine];
}

Yes, my mistake - no one is retaining the string when it is removed from the array, so it gets destroyed. Reverse the order of these two lines.

Code:
        //set the text of the textview first -  the textview will
        //retain the string if it needs it.
        [textview10 setText:chosenLine];

        //we're done with the string, now we can remove it from the array.
	[line10Array removeObjectAtIndex:chosenNumber];
